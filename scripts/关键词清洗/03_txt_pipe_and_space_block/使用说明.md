# TXT文件批量处理工具 - 使用说明

## 功能说明

本脚本可以批量处理TXT文件，执行以下操作：

1. **查找TXT文件**：自动查找指定目录下的所有.txt文件（包括子目录）

2. **处理相邻行**：
   - 检查相邻两行是否有一行仅包含竖线（|），另一行是空行或仅包含空格
   - 将符合条件的相邻两行替换为一个空行

3. **压缩连续空行**：
   - 检查是否存在3行或更多的连续空行
   - 将所有连续3行及以上的空行压缩为2个空行

## 使用方法

### 方法1：命令行运行

```bash
python txt_processor.py
```

然后按照提示：
1. 输入TXT文件所在的目录路径
2. 输入处理结果的输出目录路径
3. 等待处理完成

### 方法2：双击运行（Windows）

直接双击 `txt_processor.py` 文件，然后按照提示操作。

## 输入输出示例

### 输入路径示例
- Windows: `C:\Users\YourName\Documents\txt_files`
- Linux/Mac: `/home/username/documents/txt_files`
- 相对路径: `./input_files` 或 `input_files`

### 输出路径示例
- Windows: `C:\Users\YourName\Documents\processed_txt`
- Linux/Mac: `/home/username/documents/processed_txt`
- 相对路径: `./output_files` 或 `output_files`

## 处理示例

### 示例1：相邻行处理

**处理前：**
```
这是第一行
|||
   
这是第四行
```

**处理后：**
```
这是第一行

这是第四行
```

### 示例2：连续空行压缩

**处理前：**
```
这是第一行




这是第六行
```
（中间有4个空行）

**处理后：**
```
这是第一行


这是第六行
```
（中间压缩为2个空行）

## 注意事项

1. **文件编码**：脚本使用UTF-8编码读写文件，请确保您的TXT文件是UTF-8编码
2. **路径要求**：输入路径必须存在，输出路径会自动创建
3. **目录结构**：输出文件会保持与输入相同的目录结构
4. **备份建议**：处理前建议备份原始文件

## 常见问题

**Q: 脚本支持哪些操作系统？**
A: Windows、Linux、Mac都支持。

**Q: 如果输出目录不存在会怎样？**
A: 脚本会自动创建输出目录。

**Q: 原始文件会被修改吗？**
A: 不会，原始文件保持不变，处理结果保存在输出目录。

**Q: 支持子目录吗？**
A: 支持，脚本会递归查找所有子目录中的TXT文件。

**Q: 什么是"仅包含竖线"的行？**
A: 指一行中只有竖线字符（|），可以是一个或多个，如：`|`、`|||`、`|||||`

**Q: 什么是"空白行"？**
A: 包括完全空的行、只有半角空格的行、只有全角空格（　）的行。
